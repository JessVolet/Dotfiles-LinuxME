# Quickshell Architecture — Restructured Layout

## Project Structure

```
~/.config/quickshell/
├── shell.qml                          ← Main entry point (boots Registry only)
├── core/                              ← Global singletons & base components
│   ├── qmldir                         ← Module exports
│   ├── Theme.qml                      ← Color/style singleton
│   ├── GlobalState.qml                ← Module visibility state (NEW)
│   ├── Clock.qml                      ← Time singleton
│   ├── DPI.qml                        ← Scaling singleton
│   ├── Perf.qml                       ← Performance flags
│   ├── Workspaces.qml                 ← Workspace management
│   └── RetroBox.qml                   ← Reusable container component
├── plugins/                           ← Module implementations
│   ├── qmldir                         ← Module exports
│   ├── Registry.qml                   ← Registry (NEW: globalModules + screenModules)
│   ├── TopBar.qml                     ← Top panel (screen-scoped)
│   ├── Launcher.qml                   ← System menu (global, on-demand)
│   ├── Dashboard.qml                  ← Component catalog (global, on-demand)
│   ├── ControlCenter.qml              ← Settings center (global, on-demand) (NEW)
│   └── RiceManager.qml                ← Theme/Wallpaper manager (global, on-demand) (REDESIGNED)
└── modules/                           ← (Reserved for future modular components)
```

## Module Classification

### Global Modules (shared singleton-like, visible when needed)
- **ControlCenter**: System settings & controls
- **Dashboard**: UI component catalog/showcase
- **Launcher**: Quick access menu (System/Terminal/Files/Config)
- **RiceManager**: Theme, Wallpaper, Mode management

### Screen Modules (per-display instances)
- **TopBar**: Main panel (status, workspaces, clock, UI_CATALOG button)

## Visibility Management

### Old Architecture
- ❌ Global state flags scattered across files
- ❌ No centralized visibility logic
- ❌ Dashboard & ControlCenter shared same flag

### New Architecture
- ✅ Centralized in `core/GlobalState.qml`
- ✅ Module toggle functions: `toggleModule()`, `setModuleVisible()`, `isModuleVisible()`
- ✅ Each module has its own flag: `dashboardVisible`, `controlCenterVisible`, `launcherVisible`, `configVisible`
- ✅ Helper: `closeTransientModules()` to hide all overlays

## Rendering Pipeline

```
shell.qml (entry)
    ↓
plugins/Registry.qml
    ├─→ [Global Modules Loader]
    │   ├─ ControlCenter.qml (visibility: Core.GlobalState.controlCenterVisible)
    │   ├─ Dashboard.qml (visibility: Core.GlobalState.dashboardVisible)
    │   ├─ Launcher.qml (visibility: Core.GlobalState.launcherVisible)
    │   └─ RiceManager.qml (visibility: Core.GlobalState.configVisible)
    │
    └─→ [Variants over Quickshell.screens]
        └── [Screen Modules Loader per-screen]
            └─ TopBar.qml (screen-bound)
```

## State Flow

```
User Action (e.g., click "Configuración" button)
    ↓
Calls: Core.GlobalState.toggleConfig()
    ↓
Sets: Core.GlobalState.configVisible = true
    ↓
RiceManager.qml (visible binding) → becomes visible
    ↓
User interacts with panel
    ↓
Calls: window.applyTheme("theme-name")
    ↓
Executes: ~/scripts/rice-manager.sh theme theme-name
    ↓
Manager regenerates Theme.qml, applies Ghostty theme, updates wallpaper
    ↓
Quickshell re-renders with new colors (Theme.qml hot-reload)
```

## Component Reuse

### ThemeButton (in RiceManager.qml)
```qml
component ThemeButton: Core.RetroBox {
    property bool isActive: false
    // Show theme name + visual state
}
```

### ModeButton (in RiceManager.qml)
```qml
component ModeButton: Core.RetroBox {
    property bool isActive: false
    // Light/Dark mode selector
}
```

### ActionButton (in RiceManager.qml)
```qml
component ActionButton: Core.RetroBox {
    property bool enabled: true
    property var onAction: () => {}
    // Generic action with disabled/enabled state
}
```

## Integration Points

- **rice-manager.sh**: Backend for theme/wallpaper management
- **Ghostty**: Terminal theme (generated by manager)
- **swaybg**: Wallpaper display backend
- **Thunar**: File manager (opened for file selection)
- **Sway**: Compositor
